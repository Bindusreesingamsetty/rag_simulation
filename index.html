What is RAG?

Retrieval Augmented Generation (RAG) is a technique that allows Large Language Models to answer questions using external knowledge instead of relying only on what they were trained on.

Instead of asking the LLM directly, we first retrieve relevant documents from a knowledge base and include them in the prompt.

How Embeddings Work

Embeddings convert text into vectors (lists of numbers) so machines can measure semantic similarity.

Pizza → [0.12, 0.91, 0.44, 0.33] Biryani → [0.11, 0.88, 0.41, 0.36] Car → [0.82, 0.05, 0.11, 0.22]

Pizza and Biryani vectors are closer, so the system knows they are similar concepts (food).

Example Query Flow

User question:

"What food is famous in Japan?"

Vector search retrieves:

Sushi is a Japanese seafood dish.

The LLM then receives this prompt:

Context: Sushi is a Japanese seafood dish. Question: What food is famous in Japan? Answer using the context above.

Typical RAG Code Architecture

# 1. Load documents docs = load_documents() # 2. Split into chunks chunks = chunk(docs) # 3. Create embeddings vectors = embed(chunks) # 4. Store in vector database vector_db.store(vectors) # 5. Embed user query query_vector = embed(query) # 6. Retrieve relevant chunks context = vector_db.search(query_vector) # 7. Send to LLM response = llm(prompt(context, query))

Why RAG is Powerful

RAG solves several problems with LLMs:

  • Reduces hallucinations
  • Allows real-time knowledge
  • Works with private company data
  • Improves factual accuracy
